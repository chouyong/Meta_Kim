## Fixed

### Item 1 — ECC plugin install failure (NEW; source-of-truth verified)

- **Symptom (user-reported)**: `Failed to install plugin 'ecc@ecc' to existing marketplace 'ecc'` on macOS during `node setup.mjs` (plugin install pipeline).
- **Root cause (Fetch evidence, 7 sources)**:
  - Local current HEAD marketplace: `C:/Users/Kim/.claude/plugins/marketplaces/ecc/.claude-plugin/marketplace.json` → `name: ecc, plugin name: ecc, version: 2.0.0-rc.1`. `ecc@ecc` works here.
  - Local legacy cache: `.../marketplaces/everything-claude-code/.claude-plugin/marketplace.json` → `name: everything-claude-code, plugin name: everything-claude-code`. `ecc@ecc` does NOT resolve here.
  - User's macOS had registered marketplace id `ecc` (per `MARKETPLACE_URLS`) but pulled legacy plugin manifest; resolver couldn't find plugin name `ecc` in legacy variant.
- **Fix A — defensive spec**: `config/skills.json:105` claudePlugin `ecc@ecc → everything-claude-code@ecc`. Works against BOTH variants because legacy variant exposes the plugin under `everything-claude-code` name and current variant accepts old name as alias.
- **Fix B — force refresh**: `scripts/install-global-skills-all-runtimes.mjs` gains marketplace refresh loop (`claude plugin marketplace update <id>`) between marketplace registration and plugin install loop. Stale caches now get refreshed first.

### EB-008 (HIGH) — workerExecutionEvidence silent-success semantics

v2.2.5 contract required non-empty `actualOutput`. Silent-success commands (`node --check foo.mjs`, `tsc --noEmit`, `JSON.parse(...)` via `node -e`) produce exit-code 0 + empty stdout, creating pressure toward fabrication-adjacent placeholders like `"EXIT_OK\n"`.

- **Schema**: `config/contracts/workflow-contract.json::workerTaskPacket.workerExecutionEvidenceField` adds:
  - `successMarkerFormat` enum (`stdout-text` | `exit-code-only` | `json-output`), required field
  - `exitCode` (integer, required when `successMarkerFormat="exit-code-only"`)
  - `commandRanAt` (ISO 8601 timestamp, required when `successMarkerFormat="exit-code-only"`)
- **Reviewer rule**: `canonical/agents/meta-prism.md::Rule 16` silent-success extension. Conditions for `exit-code-only` acceptance: `successMarkerFormat="exit-code-only"` AND `exitCode=0` AND `commandRanAt` set.
- **Cross-file consistency**: Rule 16 references the EXACT same enum names as the contract (`stdout-text` / `exit-code-only` / `json-output`).

### EB-010 (MEDIUM) — sibling schema style normalized

`workflow-contract.json::workerTaskPacket` had three sibling fields with inconsistent shapes:

- `verifyStepsField` — `type: "array"` + `items: {...}` (standard JSON Schema)
- `fileCompletionListField` — `type: "array"` + `items: {...}` (standard)
- `workerExecutionEvidenceField` — `fieldType: "array"` + `itemSchema: {...}` + top-level `enforcement` block (heterogeneous)

v2.3.0 reshapes `workerExecutionEvidenceField` to match siblings:

- Top level: `type: "array"` + `items: {...}` + `_meta: {...}` sidecar
- `_meta.closes` lists `[EB-005, EB-008, EB-010]`
- `_meta` is outside validator surface (standard convention; Ajv default behavior ignores it)

## Added

### EB-003 (MEDIUM, user-selected Option D) — Meta_Kim ECC batching wrapper hook

`canonical/runtime-assets/claude/hooks/ecc-permission-cache-wrapper.mjs` (NEW, ~121 lines, 3.6KB):

- PreToolUse hook
- Cache key: `SHA256(session_id || file_path)` (per Warden C10: no global state pollution)
- TTL: 5 minutes
- Scope: per session + per file (not global, not persistent)
- Storage: `os.tmpdir()` (no project/user-home write)
- Behavior: Cache hit within TTL → return `permissionDecision="allow"` with reason. Otherwise pass through.
- Graceful: cache write/read failure is non-fatal.
- Not yet wired into `.claude/settings.json` (canonical-only landing); registration is a v2.3.x follow-up decision.

**Reviewer F1 (LOW, EB-011 backlog)**: docstring claims "batch the related ECC plugin install side-effects" but implementation only writes a per-(session,file) cache marker. It is closer to a permission auto-allow cache than true side-effect batching. v2.3.x decision: rename to `ecc-permission-cache-wrapper.mjs` OR strengthen behavior. Not a v2.3.0 blocker. — closed in v2.3.0.1 via rename + docstring alignment
