# v2.3.1 — read_only_verifier slot + dispatchChain auto-append + preDecisionOptionFrame normalization

Minor release closing three carry-forward backlog items from v2.3.0 with a coordinated unfreeze of the spine-state + enforce-agent-dispatch hook layer.

## Why this release

v2.3.0 deferred three items to v2.3.1 because they required hook-layer changes (Warden-frozen surfaces):

- **EB-002 (HIGH)** — Reviewer subagents dispatched in `review` / `meta_review` / `verification` stages cannot independently re-run verify commands (`git diff`, `npm run meta:check`, `node --check`). The PreToolUse hook denies all Bash invocations in these stages because the stage-requirements gate fires before the readonly Bash whitelist.
- **HOOK-INFRA-001 (LOW)** — `recordDispatch(state, agentName, metaName)` does not auto-append the `ownerAgent` of a matched `workerTaskPackets[]` packet to `dispatchChain[currentStage]` when dispatching to a non-governance subagent (e.g., `general-purpose`). This forces hand-edits to populate audit trails.
- **EB-004 (LOW)** — `choiceSurfaceState` had two valid locations in spine state (top-level vs `preDecisionOptionFrame.choiceSurfaceState`), violating PRIN-02 single-source-of-truth. The hook reads only the top-level location; the frame-internal value was confusingly silent.

## Fixed

### EB-002 (HIGH) + HOOK-INFRA-001 (LOW) — read_only_verifier capability slot + dispatchChain auto-append

Consolidated into one coherent fix per RFC v2 (`docs/v2.3.1-rfc-EB-002-read-only-verifier.md`):

- **`canonical/runtime-assets/shared/hooks/spine-state.mjs`**:
  - `STAGE_META_AGENT_MAP` entries for `review`, `meta_review`, `verification` now carry `readOnlyVerifierEnabled: true` + `readOnlyVerifierCommands` (21-command default whitelist: `git diff`, `git log`, `git show`, `git status`, `git rev-parse`, `git tag --list`, `npm run meta:check`, `npm run meta:check:runtimes`, `npm run meta:check:sync-coverage`, `npm run meta:validate`, `node --check`, `node -e`, `node --test`, `gh release view`, `gh pr view`, `ls`, `cat`, `head`, `tail`, `wc`).
  - `recordDispatch(state, agentName, metaName, toolInput)` now also appends to a new `dispatchChain[stage]_supplementary[]` field when a non-governance owner matched via `workerTaskPackets[].ownerAgent`. Audit trails are no longer governance-only.

- **`canonical/runtime-assets/claude/hooks/enforce-agent-dispatch.mjs`**:
  - New read_only_verifier gate inserted between `checkCapabilityNodeBindings` and the stage-requirements check. For Bash tool calls in stages where `readOnlyVerifierEnabled: true`, the gate matches whitelist prefixes and exits 0 immediately. Non-matching commands fall through to existing checks (which may deny).
  - Call site for `recordDispatch` updated to pass `toolInput` so the workerTaskPacket lookup can resolve.

### EB-004 (LOW) — preDecisionOptionFrame nesting normalization

Per RFC `docs/v2.3.1-rfc-EB-004-preDecisionOptionFrame-nesting.md`:

- **`scripts/validate-project.mjs`** gains `validateSpineStateChoiceFieldLocations()` (warn-only in v2.3.1). Scans `.meta-kim/state/*/spine/spine-state.json`; warns if `preDecisionOptionFrame.{choiceSurfaceState,solutionChoiceState,choiceGateSkip}` is set (must be top-level only). Will harden to FAIL in v2.4.0.
- **`scripts/migrate-spine-state-eb004.mjs`** (NEW) — idempotent migration helper. Promotes frame-nested copies to top-level when missing; explicit abort on disagreement.
- **`canonical/skills/meta-theory/SKILL.md`** + **`canonical/skills/meta-theory/references/dev-governance.md`** — canonical-location note added.
