# Decision Pattern Catalog

`config/governance/decision-pattern-catalog.json` stores Meta_Kim's internal decision data for Critical, Fetch, Thinking, and Review.

It was created to absorb useful reference ideas without turning another project into a dependency. Reference material can inform Meta_Kim, but it does not become an owner, weapon, skill, or invocation path unless explicitly adopted through the dependency process.

## Stage Placement

- Critical: lock the real intent, subject, current state, target state, success criteria, non-goals, and blocking unknowns.
- Fetch: label evidence as confirmed, user-provided, inferred, or unconfirmed, and bind it to decision impact.
- Thinking: compare paths and define the minimum test with pass/kill signals.
- Review: decide whether a stage actually passed, needs revision, should be killed, or must return to evidence.

## Pass / Fail

- Pass: pattern is stage-bound and owned by the correct Meta_Kim governance owner.
- Pass: pattern produces verifiable packet fields.
- Fail: reference material becomes an external dependency by accident.
- Fail: decision language replaces evidence or verification.
