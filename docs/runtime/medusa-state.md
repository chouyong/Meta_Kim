# medusa AI-context scan — runtime state schema

This document describes the local state directory used by the medusa
AI-context content scan. Files here are **runtime-local**: generated by
hooks, never committed (covered by `.gitignore`'s `.meta-kim/` rule).

## Location

```
.meta-kim/state/<profile>/medusa/
```

`<profile>` defaults to `default` and can be overridden with the
`META_KIM_PROFILE` environment variable.

## Files

### `queue.jsonl`

Append-only log of files the PostToolUse hook (`medusa-postscan-enqueue.mjs`)
or the Stop git-diff fallback wants the worker to scan.

| Field         | Type    | Notes                                                    |
|---------------|---------|----------------------------------------------------------|
| `id`          | string  | UUID per record (or `stop-fallback-<ts>-<n>`)            |
| `file`        | string  | Absolute file path, forward-slashed                      |
| `sha256`      | string  | SHA-256 of file contents at enqueue time                 |
| `mtimeMs`     | number  | mtime in ms (from `fs.statSync`)                         |
| `size`        | number  | Byte size at enqueue time                                |
| `status`      | string  | `pending` → `scanned` / `skipped` / `failed`             |
| `enqueuedAt`  | string  | ISO timestamp                                            |
| `sessionId`   | string  | Host session id (Claude/Codex/Cursor) or null           |
| `tool`        | string  | Triggering tool name (`Edit` / `Write` / etc.)          |
| `origin`      | string  | `stop-diff-fallback` when added by Stop hook fallback   |
| `scannerVersion` | string | medusa version that scanned this record (after status change) |
| `lastAttemptAt`  | string | ISO timestamp of most recent worker attempt              |
| `lastError`      | string | Error string when `status === "failed"`                  |
| `skipped`        | string | `not-applicable` / `missing` when `can_scan` filtered    |

**Status flow:**

```
            ┌─ scanned   (medusa returned issues + appended to findings.jsonl)
pending ─→ ─┼─ skipped   (can_scan said "not an AI-context file")
            └─ failed    (medusa import / scan exception; lastError populated)
```

`worker.mjs` rewrites the file in place after each drain, replacing each
record's status. The file is small (one line per pending intent) and the
rewrite is atomic via `<file>.tmp` + rename.

### `findings.jsonl`

Append-only scan results. One record per scanned file. Compact form
intended for surface hooks; full unbounded issue list lives in
`findings-full/<id>.json`.

| Field             | Type    | Notes                                                   |
|-------------------|---------|---------------------------------------------------------|
| `id`              | string  | Matches the queue record id                             |
| `file`            | string  | Absolute path                                           |
| `sha256`          | string  | sha at scan time (matches the queue record's sha)       |
| `scannedAt`       | string  | ISO timestamp                                           |
| `scannerVersion`  | string  | e.g. `2026.6.0`                                         |
| `scanMs`          | number  | Wall-clock duration in ms                               |
| `summary`         | object  | `{CRITICAL, HIGH, MEDIUM, LOW, INFO}` integer counts    |
| `issues`          | array   | Up to `META_KIM_MEDUSA_MAX_ISSUES_PER_FILE` (default 50)|
| `issuesTruncated` | number  | Count of issues dropped beyond the cap                  |
| `sessionId`       | string  | Originating host session                                |
| `hasFullArtifact` | boolean | `true` when the sidecar `findings-full/<id>.json` exists|

Each `issues[i]`:

| Field      | Type    | Notes                                                |
|------------|---------|------------------------------------------------------|
| `severity` | string  | `CRITICAL` / `HIGH` / `MEDIUM` / `LOW` / `INFO`      |
| `rule_id`  | string  | medusa rule id (e.g. `PI-RCE-002`)                  |
| `line`     | number  | 1-based line in the file                             |
| `message`  | string  | Truncated to `META_KIM_MEDUSA_MAX_MESSAGE_LEN` chars |

### `findings-full/<id>.json`

Per-record sidecar with the unbounded issue list. Created when `issues_full`
is non-empty. JSON object:

```json
{
  "id": "...",
  "file": ".../CLAUDE.md",
  "sha256": "...",
  "scannedAt": "...",
  "scannerVersion": "2026.6.0",
  "issues": [
    {
      "severity": "HIGH",
      "rule_id": "PI-RCE-002",
      "line": 258,
      "column": null,
      "code": null,
      "cwe_id": null,
      "message": "Backtick command execution"
    }
  ]
}
```

Use this for triage and audit. The compact `findings.jsonl` is what the
surface hooks render to humans.

### `classifications.jsonl`

Append-only sha-bound user acknowledgements. Records that an `(file, sha256)`
pair is an expected finding. New shas auto-invalidate prior acks.

| Field            | Type    | Notes                                            |
|------------------|---------|--------------------------------------------------|
| `file`           | string  | Absolute path                                    |
| `sha256`         | string  | sha at the time of classification                |
| `classification` | string  | e.g. `expected_security_doc_sample`              |
| `reason`         | string  | Free-text justification                          |
| `reviewer`       | string  | Identity of the user who acked                   |
| `createdAt`      | string  | ISO timestamp                                    |
| `revoked`        | boolean | `true` removes a prior ack for the same `(file, sha256)` |

### `worker.pid`

Plain text PID file. Acquired by `medusa-worker.mjs` on startup.

If the file exists and the PID is alive, a second worker exits immediately
("another worker pid alive, exit"). If the PID is dead (or the file is
stale), the new worker takes over.

Removed on graceful exit.

### `worker.log`

Short progress trace. One line per state transition (`drain start`,
`drain done`, `worker exit`, errors). Useful for diagnosing why a queued
file did not produce a finding. Not rotated; safe to delete manually.

### `last-notice.json`

`medusa-findings-surface.mjs` uses this to dedupe `UserPromptSubmit`
injections. Only re-injects when the findings fingerprint has changed.

```json
{
  "lastFingerprint": "<JSON>",
  "lastSurfacedAt": "<ISO>",
  "event": "user-prompt"
}
```

The fingerprint is built from `(file, sha256, CRITICAL, HIGH, MEDIUM)` so
both content drift and severity changes trigger re-injection.

## Trigger surface

The PostToolUse enqueue hook coarse-filters file paths against
`AIContextScanner.AI_CONTEXT_FILES + AI_CONTEXT_DIRS`. The Python helper
calls `scanner.can_scan(path)` for the authoritative decision — files that
don't pass `can_scan` are recorded as `status: skipped, skipped: not-applicable`
and produce no finding.

The Stop hook also runs a `git diff --name-only HEAD` + `git ls-files
--others --exclude-standard` fallback to catch writes that bypassed
PostToolUse (e.g. Codex `apply_patch` payloads without a per-file path).

## Control variables

| Env var                              | Default | Purpose                                      |
|--------------------------------------|---------|----------------------------------------------|
| `META_KIM_MEDUSA_SCAN`               | `warn`  | `off` / `warn` / `block` (block reserved)    |
| `META_KIM_MEDUSA_PYTHON`             | `python`| Python interpreter to spawn                  |
| `META_KIM_PROFILE`                   | `default` | State subdirectory                         |
| `META_KIM_HOOK_RUNTIME`              | autodetect | Force runtime (`claude`/`codex`/`cursor`) |
| `META_KIM_MEDUSA_WORKER_IDLE_MS`     | `240000`| Worker idle exit threshold                   |
| `META_KIM_MEDUSA_MAX_ISSUES_PER_FILE`| `50`    | Compact `issues` array cap                   |
| `META_KIM_MEDUSA_MAX_MESSAGE_LEN`    | `240`   | Per-issue message length cap (characters)    |
| `META_KIM_MEDUSA_FORCE_RESCAN`       | unset   | When `1`, ignore findings dedup and re-scan |
| `META_KIM_MEDUSA_STOP_FALLBACK_BUDGET_MS` | `1500` | Time budget for the Stop git-diff fallback |
| `META_KIM_MEDUSA_STOP_FALLBACK_MAX_FILES` | `200`  | Cap on files enqueued by the fallback   |
| `META_KIM_MEDUSA_FULL_RETENTION_DAYS`| `14`    | findings-full sidecar TTL in days            |
| `META_KIM_MEDUSA_FULL_MAX_FILES`     | `500`   | findings-full file count cap                 |
| `META_KIM_MEDUSA_FULL_MAX_MB`        | `50`    | findings-full total size cap (MB)            |

## Cleanup policy

The state directory is local and disposable. Safe ways to reset.

### POSIX (bash/zsh)

Full reset — next session rebuilds empty queue + findings as files are touched:

```bash
rm -rf .meta-kim/state/<profile>/medusa/
```

Forget all prior findings while keeping any in-flight queue:

```bash
rm -f .meta-kim/state/<profile>/medusa/findings.jsonl
rm -rf .meta-kim/state/<profile>/medusa/findings-full/
```

Recover from a stale pidfile after a crashed worker — the next enqueue or
`--once` invocation will start a fresh worker:

```bash
rm -f .meta-kim/state/<profile>/medusa/worker.pid
```

### Windows (PowerShell)

```powershell
# Full reset
Remove-Item -Recurse -Force .meta-kim\state\<profile>\medusa\

# Forget findings, keep queue
Remove-Item -Force .meta-kim\state\<profile>\medusa\findings.jsonl
Remove-Item -Recurse -Force .meta-kim\state\<profile>\medusa\findings-full\

# Stale pidfile
Remove-Item -Force .meta-kim\state\<profile>\medusa\worker.pid
```

### Automatic retention

`findings-full/<id>.json` sidecars are pruned by the worker on start/exit.
Defaults: keep at most 14 days, 500 files, or 50 MB — whichever cap is
reached first. Override with `META_KIM_MEDUSA_FULL_RETENTION_DAYS`,
`META_KIM_MEDUSA_FULL_MAX_FILES`, or `META_KIM_MEDUSA_FULL_MAX_MB`.

`findings.jsonl` is not auto-pruned; it serves as the compact audit trail
and is small. Prune manually if it ever grows unwieldy.

## Findings vs governance

`findings.jsonl` is a **scan fact**, not a governance verdict. medusa
detects pattern matches; whether each match warrants action is a separate
decision recorded in `classifications.jsonl`. Future tooling may build a
governance layer on top, but the scan layer must not silently drop
findings — only the surface layer suppresses (via ack).
